# 1029 · 第20圈年轮

一款为即将 20 岁的朋友制作的原创网页解谜生日礼物。故事围绕一棵只长了十九圈年轮的树展开：玩家需要找回日期 `1029`、名字缩写 `LXY`、年龄 `20` 与单词 `BIRTHDAY`，让第20圈在零点闭合。

整体体验参考经典点击式解谜游戏的“观察—联想—组合—反馈”循环，以及清新手绘情感游戏的叙事节奏；剧情、谜题、界面和图形均为本项目原创，没有复制商业作品的关卡、角色立绘或原作台词。

## 当前初版

- 五步主线：`1029 → 时间线提取 LXY → 花语拼出 BIRTHDAY → 补完 20 岁年轮 → 打开生日礼物`
- 七张成长卡，从初三排到大三；猫与狗分别给出提取位置和读取方向
- 雏菊、勿忘我、向日葵、桔梗组成花语机关
- 可选“带土 / 卡卡西”双人手账彩蛋，采用原创抽象符号，不使用官方美术
- 三种正向结局，没有失败结局
- 三层提示、自动存档、声音开关、重置进度、键盘焦点与移动端适配
- 纯静态前端，无第三方运行时依赖，适合 GitHub Pages
- 内容、谜题和界面分层，方便以后补照片、语音、真实回忆和更多章节

## 最重要的可修改文件

所有容易个性化的内容集中在 [`src/content.js`](src/content.js)：

- `STORY`：标题、称呼、生日、年龄、最终祝福信、署名、未来花苞
- `TIMELINE_CARDS`：初三到大三的阶段、花、字母和记忆文案
- `FLOWERS`：花名、花语和组成 `BIRTHDAY` 的字母片段
- `PUZZLE`：答案、初始乱序、提取位置、彩蛋归类
- `ENDING_COPY`：三种结局文案

因此后续补真实故事时，不需要先改 `game.js`。例如可以把某张成长卡的 `note` 替换成真实事件，把 `displayName` 从 `LXY` 改成昵称，再替换最终祝福信。

## 谜题答案（剧透）

<details>
<summary>展开完整通关流程</summary>

1. 日历圈出 10 月 29 日，并写着“月在前，日在后”，树根密码为 `1029`。
2. 七张成长卡按 `初三 → 高一 → 高二 → 高三 → 大一 → 大二 → 大三` 排列。
3. 猫项圈给出位置 `2、5、7`，狗吊牌给出“从最早读向现在”，提取出 `LXY`。
4. 花语顺序为 `雏菊（初见）→ 勿忘我（记得）→ 向日葵（向光）→ 桔梗（守约）`，字母片段拼成 `BIRTHDAY`。
5. 十九圈旧年轮加上生日当天的新一圈，得到 `20`。
6. 礼物盒依次输入 `1029 / LXY / 20 / BIRTHDAY`。
7. 可选手账：橙色旋涡与迟到的钟属于带土，银色雷光与遮住的眼属于卡卡西。

</details>

## 项目结构

```text
.
├── index.html                  # 游戏场景、可访问性结构与原创 SVG 庭院
├── styles.css                 # 清新纸感视觉、动画与响应式布局
├── src/
│   ├── content.js             # 可个性化故事、时间线、花语与答案
│   ├── game-core.js           # 可测试的纯谜题逻辑
│   └── game.js                # DOM、状态、存档、声音与完整玩法
├── tests/game-core.test.mjs   # Node.js 内置单元测试
├── docs/game-design.md        # 初版设计与扩展路线
└── .github/workflows/ci.yml   # 自动测试
```

## GitHub Pages

1. 打开仓库 **Settings → Pages**。
2. 在 **Build and deployment** 中选择 **Deploy from a branch**。
3. 选择 `main` 和 `/ (root)`。
4. 保存后等待发布。

## 后续扩展建议

- 把真实照片做成第二章“相册枝条”，每张照片带一个轻量机关
- 为初三、高中、大学拆成独立章节，并保留当前主树作为章节地图
- 加入语音、手写信、音乐或二维码彩蛋
- 增加“未来年份”空节点，让每年生日都能继续向树上追加一圈
- 将猫狗改成可选择的提示伙伴：猫负责观察，狗负责寻找隐藏物件
