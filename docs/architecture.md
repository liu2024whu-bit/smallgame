# 架构说明

项目保持原生静态前端，适合GitHub Pages，不增加不必要的框架和构建复杂度。

- `content.js` 只保存故事内容与章节元数据。
- `puzzles.js` 只保存可测试的纯谜题规则。
- `store.js` 是唯一的本地存档入口。
- `runtime.js` 提供公共运行时能力。
- `render.js` 与 `render-chapters.js` 负责界面渲染。
- `app.js` 使用统一事件委派，不给大量按钮分别注册分散监听器。
- CSS按基础、布局、章节和响应式拆分，并共享统一设计变量。

旧版不复制到主目录，避免仓库长期堆积重复运行文件；历史通过Git分支和提交记录保留。
