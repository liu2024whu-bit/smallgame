# smallgame v1 仓库审查记录

## 当前状态

当前开发线已经从简单网页小游戏转向模块化解谜项目。

已存在核心模块：

- story：时间线与情感叙事
- puzzles：章节和谜题配置
- engine：流程、事件和密码逻辑
- state：进度管理
- ui：展示层

## 已确认设计方向

主线：

初三 → 高中 → 大一 → 大二 → 大三 → 20岁生日

每个阶段必须满足：

1. 有真实记忆承载空间；
2. 有独立谜题机制；
3. 有视觉主题；
4. 有最终回收线索。

## 后续实现要求

避免：

- 直接堆叠角色元素；
- 不相关彩蛋集合；
- 剧情和代码混合；
- 单文件承载全部逻辑。

保持：

- 绣湖式观察与组合；
- 拣爱式克制情感表达；
- 手账、底片、邮戳、年轮统一视觉语言。

## 待完成重点

1. 完整入口页面；
2. 房间探索交互；
3. 六章节真实谜题实现；
4. 图片和原创素材接入规范；
5. 自动化测试；
6. 合并 main 前完整试玩。
